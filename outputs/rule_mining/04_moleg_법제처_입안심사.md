# 법제처 법령입안심사기준 전수 룰 마이닝

당신은 한국 법령 분석 AI 엔진의 규칙 설계자입니다.
법제처 법령입안심사기준(2024년판)과 행정규칙 입안심사 기준의 핵심 심사 항목을 분석하여,
아직 구현되지 않은 모든 결함 유형을 룰로 설계해 주세요.

---

## 현재 구현된 룰 (제외)

- **S-01**: 삽입조문 (조 신설 위치 부적절)
- **S-02**: 위임 조문 구조 결함
- **S-03**: 모호한 용어·표현
- **S-04**: 열거 과다 (각 호 남용)
- **L-04** (MOL-01): 포괄위임금지
- **L-05** (MOL-04): 소급입법금지·경과조치 부재
- **E-01**: 조건·단서 중첩 (복잡도 과다)
- **E-02**: 규정 형식 혼용
- **E-03**: 아날로그 규정 (기술 발전 미반영)
- **E-04**: 동일 규정 반복 (법령 간 중복)

## 법제처 법령입안심사기준 주요 심사 항목 (미구현 유형)

### MOL-02: 규제 일몰 조항 부재
신설·강화 규제에 일몰(유효기간) 조항이 없는 경우.
행정규제기본법 제8조: 규제는 원칙적으로 유효기간 명시 권고.
- TP: 새로운 규제·의무 부과 조항 + 일몰 기한 없음
- TP: "5년 이내 재검토" 또는 "○년 ○월 ○일까지 효력 유지" 조항 없음
- FP: 항구적 제도(조직법, 기본권 조항), 형사벌칙

### MOL-03: 중복 규제 — 타 법령 동일 규율
같은 사항을 이미 다른 법령에서 규율하고 있는데 다시 규정하는 경우.
법령정합성 원칙 위반.
- TP: 다른 법령의 동일 내용을 단순 반복 (예: 행정절차법 내용을 개별법에 재규정)
- TP: 상위법 조항을 하위법에 그대로 반복
- FP: 적용 범위 명확화 목적의 확인 규정 (해석상 필요한 경우)

### MOL-05: 비례원칙 위반 — 목적 대비 과도한 수단
규제 목적에 비해 수단이 과도한 조항.
헌법 제37조②, 행정기본법 제10조(비례원칙).
- TP: 경미한 위반에 영업취소·등록말소 등 과중한 제재
- TP: 의무 1회 위반 시 즉시 최고 수준 제재
- TP: 제재 전 시정 기회 부여 없는 즉시 처분 조항
- FP: 생명·안전 관련 위반 (무관용 원칙 정당)

### MOL-06: 명확성 원칙 위반 — 수범자 예측 불가능
규정이 너무 불명확하여 수범자가 금지 행위·의무를 예측할 수 없는 조항.
- TP: "부적절한 행위", "사회 통념에 반하는 행위" 등 기준 없는 규정
- TP: "관계 기관이 필요하다고 인정하는 경우" (주관적 기준)
- TP: "그 밖에 이에 준하는 경우" + 예시 없음
- FP: 일반 조항이지만 판례·해석례로 기준이 확립된 경우

### MOL-07: 위임 입법 한계 — 하위법령 위임 범위 초과
법률에서 정한 위임 범위를 시행령·시행규칙이 초과하는 구조를 만드는 조항.
- TP: 법률 위임 없이 시행령에서 새로운 의무·제한 규정
- TP: "대통령령으로 정하는 바에 따라 제한할 수 있다" (처벌·제한 권한 하위법령에 위임)
- FP: 순수 절차·형식적 사항 위임

### MOL-08: 규제 사각지대 — 예외·면제 과다
규제 적용 예외나 면제 대상을 너무 광범위하게 설정하여 규제 실효성이 없는 조항.
- TP: "다음 각 호의 어느 하나에 해당하는 경우에는 적용하지 아니한다" + 예외 6개 이상
- TP: 예외 조항의 요건이 불명확 ("특별한 사정이 있는 경우")
- FP: 정당한 이유가 있는 예외 (소규모 사업자, 공익 목적 등)

### MOL-09: 경과조치 부실 — 기존 행위 보호 미흡
법령 개정·신설 시 기존 허가·계약·권리에 대한 경과조치가 없거나 부실한 경우.
- TP: 새 규제 적용 시 기존 허가·등록 보호 조항 없음
- TP: "시행일부터 즉시 적용" + 기존 계약·권리 보호 없음
- FP: 이미 부칙에서 경과조치 규정, 적용 제외 명시

### MOL-10: 과징금 산정 기준 불명확
과징금·부과금 부과 시 산정 기준(매출액 기준, 상한·하한, 가중·감경 사유)이 없는 조항.
- TP: "과징금을 부과할 수 있다" + 산정 기준 없음
- TP: "00억원 이하의 과징금" + 구체적 산정 방식 없음
- FP: 산정 기준이 하위법령에서 정해진 경우

### MOL-11: 행정행위 취소·철회 사유 불명확
허가·인가·등록 취소·철회 사유가 불명확하거나 과도하게 광범위한 조항.
- TP: "공익을 현저히 해치는 경우 취소 가능" (기준 없는 취소)
- TP: "관계 법령을 위반한 경우" (어떤 법령? 어느 정도의 위반?)
- FP: 취소 사유를 명확히 열거한 경우

### MOL-12: 권리 구제 수단 부재
침익적 행정처분 조항에서 불복(이의신청·행정심판·행정소송) 안내 규정이 없는 경우.
- TP: 처분 조항 + 불복 절차 규정 없음
- TP: "처분에 대한 이의가 있으면 00일 이내 이의신청 가능" 없음
- FP: 이의신청 절차가 같은 법 다른 조문에 규정

## 법령체계 정합성 심사 항목 (형식적 결함)

### STR-01: 조문 제목 누락
조문에 제목(괄호 안 표시)이 없어 가독성·검색성이 떨어지는 경우.

### STR-02: 준용 조항 과다 — 준용 연쇄
다른 법령을 준용 → 그 법령이 또 다른 법령을 준용하는 연쇄 구조.
- TP: "제○조는 제△△법 제○○조를 준용한다" + 해당 조항이 또 다른 준용

### STR-03: 조항 번호 불연속
조문 번호가 불연속적이거나 가지번호(제8조의2, 제8조의3)가 너무 많은 경우.

---

## 요청

위 MOL-02~MOL-12, STR-01~STR-03 중 **텍스트 기반으로 탐지 가능한 유형 전부**를 룰로 설계하세요.

단, STR-01~STR-03 같은 형식적 결함은 심각도 "개선" 수준으로, MOL-05~MOL-06 같은 실질적 위반은 "경고" 이상으로 처리하세요.

**JSON 형식으로 출력**:

```json
{
  "rules": [
    {
      "rule_id": "MOL-02",
      "rule_name": "규제 일몰 조항 부재",
      "category": "효율성",
      "source": "행정규제기본법 제8조, 법제처 입안심사기준",
      "tp_signals": [
        "신설 규제·의무 부과 조항 + '유효기간', '재검토 기한' 없음",
        "수익적 처분 제한·금지 조항 + 일몰 기한 없음"
      ],
      "fp_filters": [
        "항구적 조직법·기본권 조항",
        "형사벌칙 (일몰 부적절)",
        "이미 부칙에서 일몰 규정"
      ],
      "regex_hints": [
        "(?:금지|제한|의무).{0,500}(?!유효기간|재검토|일몰|한시)"
      ],
      "severity": "개선",
      "description": "신설·강화 규제에 유효기간(일몰) 조항이 없어 규제가 영구화될 위험"
    }
  ]
}
```

텍스트로 탐지 가능한 모든 항목 설계 필수. 탐지 불가능한 항목(예: 타법령과 중복 판단 불가)은 detection_scope에 "법령 간 크로스체크 필요" 명시.
