## 대체 가능한 자료 (이미 sources 폴더 보유)

감사원이 감사할 때 적용하는 **규범적 기준**은 다음 법령에 명시되어 있습니다:

| 자료 | 감사원 어떤 결함 패턴 추론 가능 |
|------|------------------------------|
| `행정기본법.md` (§8~24) | 비례·신뢰보호·평등·부당결부금지 위반 |
| `행정절차법.md` (§17~28) | 처분기준 미공표·청문 누락·이유제시 부재 |
| `행정규제기본법.md` (§5~8) | 규제법정주의 위반, 일몰 부재 |
| `공직자의이해충돌방지법.md` | 이해충돌 방지 규정 부재 |
| `공공감사에관한법률.md` | 감사 면제·회피 패턴 |
| `부패방지및국민권익위원회의설치와운영에관한법률.md` | 부패취약 요인 |

→ **감사원 처분요구의 핵심 패턴은 위 법령들에서 일탈한 조문을 탐지**하는 것과 동일합니다.

## 룰 마이닝에서 감사원 사례 부재의 대응

`02_bai_감사원_규제입법.md` 프롬프트는:
- 감사원 사례 원문 없이도 **규범적 기준**(행정기본법·행정절차법)에 기반하여 룰 패턴 추론 요청
- BAI-02~BAI-10 (이중제재·청문누락·행정지도 강제화·고시 위임 등) 13개 유형을 헌법·법령 근거로 설계 요청

이 접근은 감사원 사례를 직접 보지 않아도 **동일한 결함 패턴을 포착** 가능합니다 (감사원도 결국 같은 법령을 잣대로 감사하기 때문).
