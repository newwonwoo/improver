# 통합 룰 마이닝 프롬프트 (Claude.ai 웹 1회 붙여넣기용)

> **사용법**: 이 파일 전체를 Claude.ai 웹에 한 번에 붙여넣으면, 6개 기관 자료를 토대로 50+ 신규 룰을 한꺼번에 추출합니다.

---

당신은 한국 법령 분석 AI 엔진의 규칙 설계 전문가입니다.
아래 6개 기관의 실제 시정·제재·판례 사례 + 법령 원문을 토대로,
한국 법령 텍스트에서 자동 탐지 가능한 결함 패턴을 **빠짐없이 룰로 설계**해 주세요.

---

## 1. 현재 구현된 룰 28개 (중복 금지)

- **S-01~S-04**: 구조 결함 (삽입조, 위임, 모호, 열거)
- **F-01~F-05**: 공정성 (권리제한, 면책, 처분, 의제, 재량)
- **F-07** (FTC-02): 사업자 일방적 서비스 변경권
- **F-08** (FTC-04): 자의적 이용제한·해지기준
- **F-09** (FSS-01·05): 설명의무·교부의무 부재
- **L-01~L-03**: 적법성 (인용, 교차참조, 끊긴 참조)
- **L-04** (BAI-01·MOL-01): 포괄위임금지
- **L-05** (MOL-04): 소급입법·경과조치
- **L-06** (BAI-04): 기속처분 기한 부재
- **G-01~G-05**: 거버넌스 (예외, 인허가, 감독, 내부, 보고)
- **E-01~E-05**: 효율성 (조건, 서식, 아날로그, 차등, 제재공백)

```
제6조 (일반원칙): 신의성실 위반, 부당하게 불리, 예상하기 어려운 조항
제7조 (면책조항 금지):
  ① 고의·중과실 법률상 책임 배제
  ② 손해배상 범위 부당 제한, 위험 고객에게 떠넘기기
  ③ 담보책임 배제·제한
  ④ 견본·표시 보장 내용 책임 배제
제8조 (손해배상액 예정): 부당하게 과중한 지연손해금
제9조 (계약 해제·해지):
  ① 법률상 해제·해지권 배제·제한
  ② 사업자에게 법률 없는 해제·해지권 부여
  ③ 사업자 해제·해지 요건 완화로 고객 불이익
  ④ 원상회복 의무 고객에게 과중 부담
  ⑤ 사업자 원상회복·손배 의무 부당 경감
  ⑥ 계속 채권관계 존속기간 부당, 묵시적 연장
제10조 (채무 이행): 사업자가 일방 결정·변경, 제3자 대행
제11조 (고객 권익): 항변권·상계권 제한, 기한 이익 박탈
제12조 (의사표시 의제): 침묵을 동의로 의제, 형식 부당 엄격
제13조 (대리인 책임 가중): 대리인에게 본인 채무 연대
제14조 (소송 금지): 소제기 금지, 입증책임 가중, 관할 일방 지정
```

### 3-2. 실제 시정 사례 (분야별)
- **은행·저축은행 (2025.10, 17개 유형 60개 약관)**:
  자의적 서비스 변경, "기타 필요한 경우" 불명확 사유, 개별 통지 생략, 일방적 급부 결정, 전산장애 전면 면책, 전자금융 예금 해지 과도 제한
- **가상자산 (2021.07, 8사 15유형)**:
  약관 개정권, 이용계약 중지·해지권, 서비스 이용제한, 부당 면책
- **OTT (2021, 6사 7유형)**:
  중도 해지 환불 거부, 부당 위약금, 청약철회 제한, 자의적 요금 변경, 현금환불 거부, 사이버머니 환불불가, 회원 계정 종료 사유 불명확
- **오디오북 (2024, 5사 3유형)**:
  이용내역 없는데 환불 제한, 예치금 환급 강제, 무료→유료 자동 전환
- **게임 (10사 14유형)**:
  아이템 선물 청약철회 거부, 청약철회 제한, 위법행위 배상권 침해, 이용자 제재 사전통지 부재, 게임 내 교신 무제한 열람, 미성년자 포괄 동의
- **결혼 (24개 예식장 + 18개 결혼준비대행 6유형)**:
  계약해제 시점 무관 계약금 거부 → 90일 전 통보 시 전액 환급
- **공연·티켓 (2024-2025, 19개 4분야 9유형)**:
  부당한 환불 제한, 사업자 부당 면책, 이용자 권리행사 제한 (탈퇴를 전화로만), 게시물 일방 삭제
- **헬스장 (표준약관)**: 휴·폐업 14일 전 통지

## 9. 요청

위 6개 기관 자료를 토대로, **현재 구현된 28개 룰과 중복되지 않는 모든 신규 룰**을 설계해 주세요.
예상 신규 룰 수: 약 50~60개

각 룰은 § 2의 JSON 포맷으로 출력하세요. 마크다운·코드블록 외부에 설명문 추가 금지.
출력은 단일 JSON 객체 (`{"rules": [...]}`) 만.

**룰 설계 시 주의**:
1. `regex_hints` 는 한국어 법령 텍스트에 실제 나타나는 표현 기반
2. `fp_filters` 는 정상 입법에서 오인될 수 있는 케이스 명시
3. `detection_scope` 는 단일 조문/법령 전체/타 법령 크로스체크 중 명시
4. 카테고리는 5축 (구조/공정성/적법성/거버넌스/효율성) 중 1개로 분류
5. 사례 수가 많은 패턴은 `severity`를 "경고" 이상으로

**예시 응답 형태**:
```json
{
  "rules": [
    {
      "rule_id": "FTC-REFUND-LIMIT",
      "rule_name": "환불 제한 — 일체 거부 또는 예치금 환급 강제",
      "category": "공정성",
      "source": "OTT 6사·오디오북 5사·공연 19사 시정 사례",
      "tp_signals": ["중도 해지 시 환불 불가", "예치금으로 환급", "결제 주기 내 환불 거부"],
      "fp_filters": ["사업자 귀책 시 환불 가능 명시", "법정 청약철회 7일 이내 환불 보장"],
      "regex_hints": ["환불.{0,30}(?:불가|아니|않)", "예치금.{0,20}환급", "(?:현금|결제수단).{0,30}환불.{0,20}(?:거부|제한)"],
      "severity": "경고",
      "description": "소비자의 환불 권리를 일체 거부하거나 결제수단이 아닌 예치금·사이버머니로만 환급하도록 한 조항"
    }
  ]
}
```
