# 산출물 기준 설계 Gap 분석 + 보완 + 패턴화

## Round 2: Gap 해소 — 설계서 보완

### G-01 + G-12: 분석 통계를 종합 진단 탭에 추가

**현재:** 한 줄 요약 + 레이더 + 문제현황(8건)
**변경:** 한 줄 요약 전에 통계 카드 4장 추가

```
[카드 1] 총 조문: 43개 → 이슈 조문: 28개 (65%)
[카드 2] 총 발견: 42건 (이 리포트에는 주요 8건을 상세 수록)
[카드 3] 유사 제재 사례: 9건 (감사원 4, 국감 3, 경실련 1, 뉴스 1)
[카드 4] 종합 등급: 심각 (83.1점)
```

**설계서 §3.3 Section 1 보완:**
"핵심 지표 카드"의 라벨을 기술 용어에서 쉬운 말로 전환하는 규칙 추가.

```
설계서 원문: "발견 밀도 — 100조문당 12.4건"
변경: "이슈가 있는 조문 — 43개 중 28개 (65%)"

설계서 원문: "커버리지 — 23/23패턴"
변경: "분석 범위 — 모든 항목 점검 완료"
```

→ **패턴화 규칙 P-01**: 지표 카드의 라벨은 전문가 용어 대신 "~가 ~이다" 형태의 평서문으로 작성한다.

### G-02: 카테고리별 진단 → "분야별 상세" 섹션

**현재:** 레이더 차트만 있고 카테고리를 펼쳐볼 수 없음
**변경:** 종합 진단 탭 하단에 카테고리별 접이식(accordion) 섹션 추가

```
[관리·감독 — 12건] ▼
  ├ 수탁기관 내부통제 부재 (심각)
  ├ 감독 범위·기준 불명확 (경고)
  ├ 보고 주기·양식 미정 (경고)
  └ ... (접으면 건수만 표시)

[실무 효율 — 14건] ▼
  ├ 서면 신청만 허용 (경고)
  ├ 조건 중첩 (검토)
  └ ...
```

→ **패턴화 규칙 P-02**: 카테고리별 진단은 "이름 + 건수 + 최고 등급 색상"으로 한눈에 보이게 하되, 상세는 접이식으로 숨긴다.

### G-03: 교차 패턴 표시

**현재:** 이슈가 개별적으로만 표시
**변경:** 동일 조문에 2개 이상 이슈가 걸린 경우 "⚠ 복합 문제" 배지 추가

설계서 §2.6에 따르면:
- 3패턴 동시 → "개별 수정보다 조문 재설계 권고"
- 4패턴 이상 → "구조적 결함, 분리 입법 검토"

산출물에서: 주택도시기금법의 경우 동일 조문 다중 패턴이 있는지 확인 필요.
현재 8건 이슈에서 동일 조문은 없으나, 전체 42건에서는 있을 수 있음.

→ **패턴화 규칙 P-03**: 동일 조문에 2건 이상 이슈가 걸리면 이슈 카드에 "🔗 이 조문에 다른 문제도 있습니다 (N건)" 링크를 표시한다.

### G-04: S-02 위임 검증 관점 재작성

**현재 이슈 3번:**
> "시행령에 빠진 위임 사항이 있어 조문이 작동하지 않음"

이것은 맞는 관점이지만, 설계 변경(S-02+L-04 통합)이 반영되지 않음.

**변경 포인트:**
이슈 3번의 what/why에 "위임 자체는 정상, 문제는 하위법령 미이행"을 명확히.

```
현재 what: "법률이 '대통령령으로 정한다'고 위임한 사항 중 일부가 시행령에 아직 규정되지 않았습니다."
보완: "법률이 시행령에 위임한 것 자체는 정상입니다. 문제는 위임받은 시행령이 해당 사항을 빠뜨렸다는 것입니다."
```

→ **패턴화 규칙 P-04**: 위임 관련 이슈는 반드시 "위임 자체 ≠ 문제"를 첫 문장에서 명확히 한다.

### G-05: 사내규정 체크리스트 통합 뷰

**현재:** 각 이슈의 process 필드에 분산
**변경:** "개선 일정" 탭 하단에 체크리스트 섹션 추가

```
□ 기금운용 업무지침에 변경 사유·절차·심의 요건 반영
□ 수탁기관 내부통제기준서 신규 작성
□ 분기별 보고 일정·담당자·양식 업무처리규정에 명시
□ 전자신청 시스템 구축 계획 수립
□ 시행령 누락 사항 전수 점검
□ 감독 대응 체계(자료 준비, 담당자 지정) 규정 마련
□ 대출심사 매뉴얼 요건 1단계·2단계 분리 정리
□ 기금 집행 시 '직접 관련 사업' 확인 체크리스트 도입
```

→ **패턴화 규칙 P-05**: 모든 이슈의 process 필드를 자동으로 수집하여 "사내규정 반영 체크리스트"를 생성한다. 리포트 하단에 체크박스 형태로 통합 표시.

### G-06: 기관 판단기준 표시

**현재:** 제재 사례만 표시, 판단 "기준"은 미표시
**변경:** 사례 카드 상단에 1줄짜리 "근거 법령/기준" 추가

```
📐 근거: 감사원법 §33 (감사 결과 처분 요구), 공공기관운영법 §48 (경영실적 평가)
```

→ **패턴화 규칙 P-06**: 제재 사례에는 반드시 "이 기관이 이 처분을 할 수 있는 법적 근거" 1줄을 함께 표시한다.

### G-08: 표지 보완

**변경:** 법령 종류(법률) + 시행일 추가

```
현재: "2026년 5월 17일 · 총 43개 조문 · 최종 개정 2025.1.7"
변경: "법률 · 시행 2015.7.1 · 최종 개정 2025.1.7 · 총 43개 조문"
```

→ **패턴화 규칙 P-07**: 표지 메타정보는 "법령종류 · 시행일 · 개정일 · 조문수" 순서로 고정한다.

### G-09: fix_type 시각화

**변경:** Before/After 카드에 아이콘으로 수정 유형 표시

```
✂️ 삭제형: 문제 표현을 삭제
🔄 교체형: 문제 표현을 대체
📎 단서추가형: 조문에 단서 추가
➕ 항신설형: 새 항 추가
📋 하위법령형: 법률은 안 고치고 시행령/고시로 보완
```

→ **패턴화 규칙 P-08**: 개선안은 5가지 수정 유형 중 하나로 분류하고, 아이콘+한 줄 라벨로 표시한다.

### G-11: 기관 연계

**변경:** 로드맵에 "관련 기관" 표시

```
[30일] 즉시 개선 — 관련: 국토교통부(감독 시정), 감사원(후속 점검)
[90일] 빠른 개선 — 관련: 법제처(시행령 정비), 기재부(기금운용계획)
```

→ **패턴화 규칙 P-09**: 로드맵의 각 단계에는 "어느 기관과 협의/보고해야 하는가"를 1줄로 표시한다.

## Round 3: 패턴화 — 리포트 생성 규칙 10개

위 Gap 해소 과정에서 추출한 규칙을 표준화한다.
이 규칙은 어떤 법령을 분석하든 동일하게 적용한다.

### 리포트 생성 패턴 (Report Generation Patterns)

```
P-01 [지표 카드] 라벨은 전문가 용어 대신 평서문.
     ✗ "발견 밀도 — 100조문당 12.4건"
     ✓ "이슈가 있는 조문 — 43개 중 28개 (65%)"

P-02 [카테고리 뷰] "이름 + 건수 + 최고등급 색상" 한눈에.
     상세는 접이식. 펼치면 이슈 목록.

P-03 [교차 패턴] 동일 조문 2건↑ 이슈 → "🔗 이 조문에 다른 문제도 N건" 표시.
     3건↑ → "조문 재설계 필요" 경고.

P-04 [위임 표현] "위임 자체 ≠ 문제"를 첫 문장에서 명확히.
     "하위법령이 빠졌다"가 핵심이지 "위임이 많다"가 아님.

P-05 [체크리스트] 모든 이슈의 process → 자동 수집 → 통합 체크리스트.
     리포트 하단에 체크박스 형태.

P-06 [기관 근거] 제재 사례에 "이 기관이 이 처분을 할 수 있는 법적 근거" 1줄 필수.
     예: "📐 근거: 감사원법 §33, 공공기관운영법 §48"

P-07 [표지 메타] "법령종류 · 시행일 · 개정일 · 조문수" 순서 고정.

P-08 [수정 유형] 개선안은 5가지로 분류 + 아이콘.
     ✂️ 삭제 / 🔄 교체 / 📎 단서추가 / ➕ 항신설 / 📋 하위법령보완

P-09 [기관 연계] 로드맵 각 단계에 "협의/보고 대상 기관" 1줄.

P-10 [사례 출처] 모든 제재 사례에 URL 링크 필수.
     기관명 + 날짜 + 기사 제목 + 클릭 가능 링크.
```

## Round 4: 설계서 반영 대상 정리

### 핵심 설계서 수정
