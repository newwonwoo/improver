# TF 새출발 설계서 — 신경망·추론 정체 돌파 (v1)

> 목표: 신경망(SLM)·추론엔진 개선 정체를 새출발로 돌파.
> 원칙: 담당자 역할 불가침. 모든 담당자는 "전원 의견조회 → 취합 → 초안 → 팀장 보고" 절차로 일한다.
> 보고: 코더·슈퍼바이저·테스터는 팀장 경유. 설계자는 핵심만 직보.

## 정체 진단 (왜 막혔나)
1. 라벨이 전부 FP (TP 0) → 신경망이 "진짜 결함" 경계를 못 배움.
2. 데이터 소스가 해석성(moleg 해석례)뿐 → 결함지향 정답이 없음.
3. 엔진이 단일조문 스캔 → 법-간 우선적용(특별법 vs 일반법) 구조적 미탐지.
4. 라벨→학습기 연결 끊김 (importer fid `rule-batch-id` ≠ 트레이너 `rule@법령`).
5. FP필터를 룰 하나씩 수동(P-META-1/1b) → 반복노동, 일반화 안 됨.

---

## 단계별 설계 (담당자 / 업무지시 / 산출물 / 완료기준 / 의존)

### S1. 결함지향 데이터 크롤러
- **담당**: 데이터수집가(주) · 설계자(스펙) · 코더(구현)
- **업무지시**: 감사원 감사결과·헌재 위헌결정·법제처 법령정비 권고 3개 소스 어댑터를 `outputs/rule_mining/sources/crawled/{bai,ccourt,moleg_revise}/`에 수집. 각 레코드에 `{법령, 조문, 결함유형, 출처, 원문}` 정규화.
- **산출물**: 소스별 크롤러 + 정규화 jsonl
- **완료기준**: 3개 소스 합산 **결함지향 케이스 ≥ 150건**, 그중 코퍼스 내 법령 매칭 ≥ 60%
- **의존**: 없음 (선행)

### S2. 라벨→학습기 연결부 수정
- **담당**: 코더(주) · 슈퍼바이저(테스트)
- **업무지시**: `phase13_import_verdict_responses.py` fid를 `{rule_id}@{primary_law}`로 통일하고 `fid_article_map[fid]=primary_article_norm` 자동등록. `RULE_CAT` 미매핑 rule은 경고 후 skip.
- **산출물**: 수정 importer + fid_article_map 자동 갱신
- **완료기준**: 임의 라벨 N건 import 시 `collect_torch_data`가 **0행 스킵 없이** N행 적재(스모크 테스트 통과)
- **의존**: 없음 (S1과 병행 가능)

### S3. 법-간 교차참조 그래프
- **담당**: 설계자(아키텍처) · 코더(구현)
- **업무지시**: `engine/graph/law_graph.py`에 "조문 주제 ↔ 타법 동일주제" 엣지 구축. 한 조문이 다른 법의 규율영역과 겹치면 `has_cross_law_overlap` 신호 생성 → R-LAW-PRECEDENCE premise에 연결.
- **산출물**: 교차참조 신호 + R-LAW-PRECEDENCE 발화 경로
- **완료기준**: 검증된 5건(관광진흥법 등) 중 **≥ 3건 신규 포착**, 전체 발화율 ≤ 0.5%(과탐 억제)
- **의존**: 없음 (독립 트랙)

### S4. TP·FP 균형 배치 재학습
- **담당**: 코더(주) · 슈퍼바이저/테스터(검증)
- **업무지시**: S1 데이터로 TP 포함 verdict 배치 export → 라벨 → S2 연결부로 적재 → `train_torch` 재학습(선형→MLP). class_weight balanced.
- **산출물**: TP 포함 학습셋 + 재학습 모델
- **완료기준**: 학습셋 **TP ≥ 30건**, holdout F1 ≥ 기존 baseline(0.612) 초과
- **의존**: S1, S2

### S5. FP필터 신호 통합·자동 튜닝
- **담당**: 코더(주) · 설계자(신호 정의)
- **업무지시**: P-META-1/1b식 수동 필터를 `enrich_*` 신호군으로 일반화하고 `tune_engine.py`가 verdict에서 임계·negation을 자동 제안(`apply_proposals --dry-run` 검토 후 적용).
- **산출물**: 신호 자동튜닝 파이프라인
- **완료기준**: 500샘플 R-DELEG-BLANKET FP **추가 ≥ 20%↓**, 과억제(진짜 결함 누락) ≤ 5%
- **의존**: S2 (라벨 흐름)

### S6. 개선안 품질 채점 루프
- **담당**: 설계자(평가설계) · 최종검수인(기준선 라벨)
- **업무지시**: 진짜 결함(TP) 조문의 Layer3 개선안을 `{정확·구체·실행가능}` 3축으로 채점. 사람 라벨 20건 기준선 + 자동채점 상관 측정.
- **산출물**: 개선안 평가 지표 + 리포트
- **완료기준**: 사람-자동 채점 상관 ≥ 0.6, 저품질 개선안 자동 플래그
- **의존**: S4 (TP 확보 후)

---

## 마일스톤 순서
```
S1(데이터) ─┐
S2(연결부) ─┼─→ S4(재학습) ─→ S6(개선안 채점)
S3(교차참조) 독립 트랙 ──────────────────┘
S5(자동튜닝)는 S2 이후 상시
```
1~2주차: S1·S2·S3 병행 → 3주차: S4·S5 → 4주차: S6 + 종합검수.

## 리스크 / 가드
- 과억제: FP 줄이려다 진짜 결함 누락 → 슈퍼바이저가 매 단계 over-suppression 지표 감시.
- 데이터 편향: 감사원/헌재가 특정 분야 편중 → 데이터수집가가 분야 분포 보고.
- 자동채점 과신: 최종검수인이 사람 기준선 없이는 승인 불가.

## S3 상세설계 — 법-간 교차참조(왜 별도 큰 작업인가)

### 문제
R-LAW-PRECEDENCE(특별법 vs 일반법 우선) 미탐지 5건(#8·12·13·31·33)을 분석한 결과:
- 우선적용 다툼은 **두 법의 관계**인데, 현 신호 `has_undefined_precedence`는 **한 조문 본문**에
  "다른 법률의 특별한 규정"류 문구가 있을 때만 켜짐.
- 실제 충돌 법령을 **조문이 인용조차 안 함**: 관광진흥법 제15조는 국토계획법(충돌상대)을 안 부름.
- 기존 `engine/graph/law_graph.py`는 **인용(citation) 그래프** — 인용 안 하면 엣지 없음 → 무력.

### 결론
정규식 확장·인용기반 휴리스틱으로는 **그 5건을 못 잡음**(자기선언·자기인용이 없음). → band-aid 금지.

### 제대로 된 설계 (구현 미완 — 별도 트랙)
1. **주제(topic) 추출**: 각 조문의 규율 대상(시설·행위·권리 등)을 키워드/임베딩으로 라벨.
2. **주제-중첩 인덱스**: 전 코퍼스에서 같은 주제를 규율하는 (법A·조, 법B·조) 쌍 그룹화.
3. **특별/일반 판정**: 한 쌍이 특별법(개별 산업·시설법) vs 일반법(국토계획·행정절차 등)이면
   `has_cross_law_overlap` 신호 → R-LAW-PRECEDENCE premise 에 OR 연결.
4. **검증**: 5건 중 ≥3 신규 포착, 전체 발화율 ≤0.5%(과탐 억제).

### 비용
전 코퍼스 주제 인덱싱 + 특별/일반 사전 구축 = 수일 규모. citation 그래프 재사용 불가(주제축 신규).
→ S1·S2·S4·S5·S6 안정화 후 착수 권장.
