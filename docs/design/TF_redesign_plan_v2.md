# TF 새출발 설계서 v2 — 증거기반 재방향 (전원 의견조회 반영)

> 작성: 설계자 취합 → 팀장 확정. v1 대비 **핵심 전제 교체**.
> 절차: 4개 레인(데이터·코더·테스터·검수) 의견조회 → 본 문서로 취합.

## v1이 틀렸던 전제
v1은 "**TP 데이터 부족**이 병목"이라 가정 → S1(결함데이터 크롤)·S4(라벨 증량) 우선.
**실측으로 반증됨**: TP를 372→512(+38%)로 늘렸는데 신경망 F1이 **0.536→0.522→0.516(하락)**.
→ **데이터량은 병목이 아니다.** 4개 레인 전원 동의.

## v2의 새 전제 (전원 합의)
1. **지금 0.52는 믿을 수 없는 숫자다.** 단일 20% 분할, 카테고리 셀 6~30개 → ±0.1 출렁. 0.536→0.516 하락이 노이즈인지 진짜인지 구분 불가.
2. 라벨링의 진짜 자산은 verdict가 아니라 **신호 후보(+51, 누적 381)**.
3. 저정밀 룰(S-01·S-02)은 라벨 100건 중 **TP 0** = 룰 정의가 코퍼스 현실과 불일치.

---

## 단계별 설계 (할 일 / 이유 / 담당 / 완료기준)

### V1. 평가 신뢰성 재구축 — **최우선 P0**
- **할 일**: `train_torch`를 `_fit_once(train_idx,test_idx)`로 분리 → stratified K-fold(5) + bootstrap 95% CI + 카테고리별 표본수 병기.
- **이유**: 측정이 노이즈면 그 위의 모든 결론(정체·회귀·개선)이 무의미. 코더 추정 반나절·저위험.
- **담당**: 테스터(설계)·코더(구현)
- **완료기준**: per-cat F1±CI 출력, n<15 "측정불가" 분리, **baseline 0.612와 동일 분할·동일 F1정의로 paired 재계산** 표.

### V2. 라벨 증량 중단 + BORDER 격리
- **할 일**: 신규 라벨링 중단(49번들 보류). BORDER 293건은 학습 제외 → 2차 검수 큐.
- **이유**: 양 병목 아님 확인. 노이즈 평가 위에 라벨 적재는 헛심(데이터수집가).
- **담당**: 데이터수집가
- **완료기준**: 학습셋에서 BORDER 제외, 중단 공지.

### V3. 모델 단순화 대조군
- **할 일**: 로지스틱(선형)+Platt/isotonic 캘리브레이션(`slm_calibrate.py` 재사용)으로 baseline 재대결. MLP와 K-fold F1 비교.
- **이유**: 2,701행에 임베딩+2층 MLP는 과사양 → 정체가 과적합 신호일 수 있음(코더).
- **담당**: 코더
- **완료기준**: 선형 vs MLP K-fold F1±CI 비교표. 선형이 비열위면 MLP 폐기.

### V4. 신호 상위 N개 leakage-safe 통합 (구 S5)
- **할 일**: 신호 381개 중 **빈도·정밀도 컷 상위 N개만** `FEATURE_NAMES`에 등록(신호별 on/off 추적). 표본 20% 사람 역검증 후 통합.
- **이유**: 라벨로 만든 신호를 그 라벨로 평가 = **순환논증**(검수인). 일괄통합은 오염신호 격리 불가. tune_engine/apply_proposals는 타깃 불일치라 경량 신설(코더).
- **담당**: 코더·설계자 / 검수인 게이트
- **완료기준**: 신호별 정밀도 역검증표, leakage 점검표 통과분만 투입.

### V5. 저정밀 룰 비활성 (삭제 아님)
- **할 일**: S-01·S-02의 **고유 적중**(다른 룰이 못 잡는 결함) 케이스 추출 → 없으면 **비활성+회귀세트 보존**. 잘되는 E/F 신호를 S용으로 역설계 이식 검토.
- **이유**: 저정밀이어도 유일검출원이면 은퇴=회귀(검수인·테스터). 삭제 불가·롤백 가능해야.
- **담당**: 설계자 / 검수인 게이트
- **완료기준**: 고유적중 목록+대체커버 매핑, recall 하한 비악화.

### V6. 최종 게이트 = 개선안 채택률 (구 S6)
- **할 일**: 정체 해제 판정은 F1·K-fold가 아니라 **전문가 블라인드 권고 채택률 + 진짜결함 적중 절대증가**.
- **이유**: 중간지표 올라도 쓸모없는 권고 가능. 실효 적중이 늘어야 진짜 해제(검수인).
- **담당**: 최종검수인
- **완료기준**: 권고 (진짜결함/오탐/사소) 3분류, baseline 대비 진짜결함 적중 절대 증가.

---

## 게이트 원칙 (검수인)
- 룰·필터 변경: **recall 비악화 AND precision 개선** (단일 F1 게이트 금지 — 과억제 은폐).
- 신호 자동적용: 출처·중복·leakage 점검표 통과분만, 신호별 on/off.
- 정체 해제 선언: ①회귀세트 recall 보존 ②신호 표본 역검증 ③전문가 채택률·진짜결함 절대증가 — **셋 다** 충족까지 보류.

## v1 → v2 변화 요약
| | v1 | v2 |
|---|---|---|
| 1순위 | 결함데이터 크롤(S1) | **평가 신뢰성(V1)** |
| 라벨 | 증량(S4) | **중단(V2)** |
| 모델 | MLP 재학습 | **선형 대조군 먼저(V3)** |
| 정체해제 기준 | F1>baseline | **개선안 채택률(V6)** |

## 마일스톤
```
V1(평가) → V2(중단) 즉시 → V3·V4·V5 병행 → V6 최종게이트
```
강등: v1의 S1(크롤)·S4(라벨증량)은 병목 아님 판명 → 보류.
