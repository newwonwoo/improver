# 카테고리별 하위 점검항목 (Sub-Check) 재설계

> 변경: 헌재(위헌 판단) 제거. 이미 만들어진 법률의 위헌 소지를 판단하는 건 이 도구의 범위 밖.
> 방법: 이전 대화에서 설계한 5대 기관 판단기준 + 실제 스캔 데이터를 기반으로 각 패턴에 하위 점검항목 추가
> 기준 기관: 법제처, 공정위, 감사원, 금감원, 권익위 (4대 + 법제처 = 5대)

## 구조 변경: 패턴 → 서브체크

```
Before: G-04 내부통제 → "있다/없다"
After:  G-04 내부통제
          ├ G-04-a 통제환경 → 있다/없다 + 구체성 점수
          ├ G-04-b 위험평가 → 있다/없다 + 구체성 점수
          ├ G-04-c 통제활동 → 있다/없다 + 구체성 점수
          ├ G-04-d 정보소통 → 있다/없다 + 구체성 점수
          └ G-04-e 모니터링 → 있다/없다 + 구체성 점수
        → "5개 중 2개만 있음 (40%)" + "가장 심각한 빈칸: 위험평가"
```

서브체크 스키마:
```json
{
  "sub_check_id": "G-04-b",
  "name": "위험평가 절차",
  "description": "기금운용 위험을 식별·평가하는 절차가 규정되어 있는가",
  "detection_keywords": ["위험평가", "위험관리", "리스크", "위험요인"],
  "found": false,
  "evidence": null,
  "agency_basis": "감사원 내부통제 가이드라인 2요소",
  "real_case": "감사원 2024.8 — HUG 위험관리체계 미흡 지적"
}
```

## 1. 구조 (S) — 4패턴 14서브체크

참조 기관: **법제처** (입안길잡이)

### S-01. 삽입조 (3서브체크)

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| S-01-a | 삽입 깊이 | "제N조의M" 파싱, depth 계산 | depth 3↑ = 경고 |
| S-01-b | 삽입 밀도 | 삽입조 비율 = inserted / total | 30%↑ = 경고 |
| S-01-c | 연속 삽입 | "제N조의2, 제N조의3, 제N조의4..." 연속 카운팅 | 연속 5↑ = 심각 |

주택도시기금법 실제: 삽입조 확인 필요 (이전 스캔에서 337건은 부투법)

### S-02. 위임 검증 (4서브체크)

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| S-02-a | 위임 목록 | "대통령령으로 정하" 키워드 추출 | 정보 수집 (등급 아님) |
| S-02-b | 하위법령 이행 | 시행령 DB에서 대응 조문 존재 확인 | 미이행 = 심각 |
| S-02-c | 포괄위임 | "필요한 사항" "기타 사항" 탐지 | 포괄 = 주의 |
| S-02-d | 위임 체인 | 법률→시행령→시행규칙→고시 다단계 확인 | 3단↑ = 경고 |

주택도시기금법 실제: 위임 53건(대통령령42+국토부령11), 포괄위임 다수

### S-03. 모호 표현 (4서브체크)

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| S-03-a | 불확정개념 | "상당한/적절한/필요한 경우" 9종 키워드 | 핵심 요건에 2↑ = 심각 |
| S-03-b | 잔여 표현 | "등" "기타" "그 밖의" | 열거 뒤 "등" 카운팅 |
| S-03-c | 재량 근거 불명확 | 의무 조항에 "~할 수 있다" + 기준 없음 | F-05와 연계 |
| S-03-d | 범위 미한정 | "~에 관한 사항" 등 범위 열린 표현 | S-02-c와 연계 |

주택도시기금법 실제: 모호표현 31건

### S-04. 열거 과다 (3서브체크)

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| S-04-a | 호 수 | 항 내부 호("1." "2." ...) 카운팅 | 20↑ = 경고, 30↑ = 심각 |
| S-04-b | 목 수 | 호 내부 목("가." "나." ...) 카운팅 | 10↑ = 주의 |
| S-04-c | 유사 항목 존재 | 호 텍스트 유사도 비교 (중복/통합 가능) | 유사 3쌍↑ = 주의 |

## 2. 공정성 (F) — 5패턴 19서브체크

참조 기관: **공정위** (약관규제법 8유형), **금감원** (소비자보호)

### F-01. 권리 제한 (5서브체크)

출처: 공정위 약관규제법 §9~§11

| ID | 점검항목 | 탐지 방법 | 기준 | 공정위 유형 |
|----|---------|----------|------|-----------|
| F-01-a | 해제·해지권 박탈 | "해지할 수 없다" "해제를 제한" | 고객 해지권 없음 = 심각 | §9 |
| F-01-b | 일방적 급부 변경 | "변경할 수 있다" + 통지 의무 없음 | 사업자만 변경 가능 = 경고 | §10 |
| F-01-c | 항변권·상계권 박탈 | "항변할 수 없다" "상계하지 못한다" | 무효 소지 = 심각 | §11 |
| F-01-d | 기한이익 상실 | "기한의 이익을 상실" + 사유 불명확 | 포괄적 상실 사유 = 경고 | §11 연관 |
| F-01-e | 구제 절차 부재 | 해당 조문 ±2조 내 이의/청문/소명 키워드 부재 | 구제수단 없음 = 심각 | — |

### F-02. 면책 조항 (3서브체크)

출처: 공정위 약관규제법 §7

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| F-02-a | 전면 면책 | "일체의 책임" + "지지 아니한다" | 고의·중과실 예외 없음 = 심각 |
| F-02-b | 부분 면책 | "책임을 지지 아니한다" (일체 없이) | 범위 불명확 = 경고 |
| F-02-c | 고의·중과실 예외 존재 | 같은 조문에 "고의 또는 중과실 제외" | 있으면 양호 |

### F-03. 처분·명령 (4서브체크)

출처: 금감원 검사제재 기준

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| F-03-a | 처분 강도 | "영업정지/인허가취소/등록취소" | 강한 처분 = 높은 점검 대상 |
| F-03-b | 사전 절차 | 해당 법령 내 "청문/의견제출/이의신청" 존재 | 없으면 = 심각 |
| F-03-c | 처분 기준표 | 별표/별도 기준 존재 여부 | 없으면 = 경고 |
| F-03-d | 비례원칙 | 처분 수위가 단일(전부 아니면 전무)인지, 단계적인지 | 단일 = 경고 |

### F-04. 의사표시 의제 (4서브체크)

출처: 공정위 약관규제법 §12

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| F-04-a | 의제 유형 판별 | "동의/승낙/이의/갱신" + "본다/간주" | 의사표시 의제 = 대상 |
| F-04-b | 사전 통지 의무 | 같은 조문에 "통지하여야/고지하여야" | 없으면 = 심각 |
| F-04-c | 기간 적정성 | 의제 전 기간 추출 (N일) | 14일 미만 = 심각, 30일↑ = 주의 |
| F-04-d | 철회 가능 | "철회할 수 있다/취소할 수 있다" | 없으면 = 경고 |

주택도시기금법 실제: 의제 2건

### F-05. 자의적 재량 (3서브체크)

출처: 금감원 검사 시 재량 남용 판단

| ID | 점검항목 | 탐지 방법 | 기준 |
|----|---------|----------|------|
| F-05-a | 주어 분류 | 행정청/사인/내부기관 판별 | 행정청만 대상 |

### G-04. 내부통제 (5서브체크)

출처: **감사원 내부통제 가이드라인 5대 요소** + **금감원 지배구조법 시행령 §19 (13개 항목)**

| G-04-b | 위험평가 | ② 리스크평가 | "위험평가/위험관리/리스크/위험요인" | 부재 = 심각 |
| G-04-c | 통제활동 | ③ 통제활동 | "승인절차/직무분리/접근통제/결재" | 부재 = 경고 |
| G-04-d | 정보소통 | ④ 정보·의사소통 | "보고체계/공시/경영공시/정보공유" | 부재 = 경고 |
| G-04-e | 모니터링 | ⑤ 모니터링 | "자체점검/경영실적평가/내부감사/모니터링" | 부재 = 심각 |

**금감원 지배구조법 시행령 §19 (13개 항목) — G-04 세부 기준:**
```
1. 업무분장 및 조직구조 → G-04-a
2. 임직원이 직무 수행 시 준수할 기준과 절차 → G-04-a
3. 경영의사결정에 필요한 정보가 효율적으로 전달될 수 있는 체계 → G-04-d
4. 임직원의 내부통제기준 준수 여부를 확인·점검·보고 → G-04-e
5. 위반 시 조치에 관한 사항 → G-04-e
6. 내부통제와 관련하여 관계 법규의 제·개정 등에 따라 필요한 사항 → G-04-a
7. 이해상충 관리에 관한 사항 → G-04-a
8. 임직원의 금융거래에 관한 사항 → G-04-c
9. 임직원의 재산상 거래에 관한 사항 → G-04-c
10. 고객에 관한 정보 등의 관리에 관한 사항 → G-04-c
11. 전산시스템 등에 대한 보안에 관한 사항 → G-04-c
12. 업무위탁에 관한 사항 → G-04-c
13. 그 밖에 내부통제 운영에 필요한 사항 → G-04-a
```

주택도시기금법 실제: 제22조 "업무지침을 정하여야 한다" — G-04-a~e 전부 미충족

## 10. 서브체크별 사유 정의 (v2 신규)

### 10.1 설계 원칙

각 서브체크에 "이게 빠지면 왜 문제인가"를 1~2문장으로 정의한다.
사유의 출처: 5대 기관 기준 + 전수 분석 통계 + 실제 제재 사례.
전문용어 금지. 결과 중심으로 서술.

구조: `[빠진 것]이 없으면 → [구체적 결과]가 발생합니다.`

### 10.2 구조 (S) 서브체크 사유

| ID | 서브체크 | 사유 |
|----|---------|------|
| S-01-a | 삽입 깊이 | 조문번호가 "제34조의2의3"처럼 깊어지면 어떤 조문을 봐야 하는지 찾기 어려워, 실무에서 잘못된 조문을 적용할 수 있습니다. |
| S-01-b | 삽입 밀도 | 법 전체의 30% 이상이 나중에 끼워넣은 조문이면, 법의 체계가 무너져 전부개정이 필요한 상태입니다. |
| S-01-c | 연속 삽입 | 같은 조문 뒤에 5개 이상 연속으로 끼워넣으면, 원래 조문과의 관계를 파악할 수 없습니다. |
| S-02-a | 위임 목록 | (정보 수집용 — 사유 불필요) |
| S-02-b | 시행령 이행 | 법이 시행령에 맡긴 사항이 시행령에 없으면, 해당 조문은 있으나마나한 상태(사문화)가 됩니다. |
| S-02-c | 포괄위임 | "필요한 사항"처럼 범위가 열려 있으면, 시행령에서 뭘 정해야 하는지 알 수 없습니다. |
| S-02-d | 위임 체인 | 법→시행령→시행규칙→고시로 3단계 이상 넘기면, 실제 규정을 찾으려면 4개 문서를 봐야 합니다. |
| S-03-a | 불확정개념 | "상당한" "적절한" 같은 표현이 의무 조항에 있으면, 지켜야 할 기준이 사람마다 달라집니다. |
| S-03-b | 잔여 표현 | "등" "기타"로 끝나면 목록이 열려 있어, 나중에 아무거나 포함시킬 수 있습니다. |
| S-03-c | 재량 근거 불명확 | "할 수 있다"만 있고 기준이 없으면, 담당자 재량으로 결과가 달라집니다. |
| S-03-d | 범위 미한정 | "~에 관한 사항"처럼 범위가 열려 있으면, 위임받은 쪽에서 임의로 범위를 정합니다. |
| S-04-a | 호 수 | 한 조문에 20개 이상 나열하면, 읽는 사람이 전체를 파악할 수 없습니다. |
| S-04-b | 목 수 | 호 안에 다시 10개 이상 세분하면, 구조가 너무 복잡해져 빠뜨리기 쉽습니다. |
| S-04-c | 유사 항목 | 비슷한 내용이 반복 나열되면, 통합하거나 유형화해야 합니다. |

### 10.3 공정성 (F) 서브체크 사유

| ID | 서브체크 | 사유 |
|----|---------|------|

| F-03-a | 처분 유형 | (정보 수집용 — 사유 불필요) |
| F-03-b | 사전 절차 | 처분 전에 소명 기회가 없으면, 오해나 착오로 부당한 처분을 받을 수 있습니다. |

| F-03-d | 비례원칙 | 처분이 "취소 아니면 무"식 단일 수위면, 경미한 위반에도 최대 제재를 받습니다. |
| F-04-a | 의제 유형 | (분류용 — LLM이 판단) |
| F-04-b | 사전 통지 | 통지 없이 침묵을 동의로 간주하면, 본인도 모르는 사이에 권리가 변동됩니다. |

| F-04-d | 철회 가능 | 한번 간주되면 돌이킬 수 없으면, 실수나 사정 변경에 대응할 수 없습니다. |
| F-05-a | 주어 분류 | (분류용 — 사유 불필요) |
| F-05-b | 발동 요건 | "필요하다고 인정하면"처럼 요건이 없으면, 아무 때나 재량을 행사할 수 있습니다. |
| F-05-c | 효과 한정 | "적절한 조치"처럼 효과가 열려 있으면, 어떤 조치든 가능해집니다. |

### 10.4 적법성 (L) 서브체크 사유

| ID | 서브체크 | 사유 |
|----|---------|------|
| L-01-a | 인용 수 | (정보 수집용) |
| L-01-b | 폐지법령 인용 | 없어진 법을 인용하면, 해당 조문의 근거가 사라진 상태입니다. |

| L-02-a | 참조 조문 존재 | 참조한 조문이 삭제되었으면, 참조 자체가 무효입니다. |
| L-02-b | 참조 조문 개정 | 참조한 조문의 내용이 바뀌었으면, 원래 의도와 달라졌을 수 있습니다. |
| L-02-c | 과도 타법 의존 | 자체 규정 없이 다른 법만 참조하면, 그 법이 바뀔 때 같이 영향받습니다. |
| L-03-a | 법령 부존재 | 인용한 법 자체가 현행법에 없으면, 규범 공백 상태입니다. |
| L-03-b | 조문 삭제 | 법은 있으나 해당 조문이 없으면, 참조가 끊긴 상태입니다. |
| L-03-c | 법령명 불일치 | 띄어쓰기·약칭 차이로 매칭 안 되면, 실무에서 혼란을 줍니다. |

### 10.5 거버넌스 (G) 서브체크 사유

| ID | 서브체크 | 사유 |
|----|---------|------|
| G-01-a | 단서 중첩 | "다만"이 3번 반복되면, 원칙이 뭔지 알 수 없습니다. |
| G-01-b | 예외 비율 | 조문의 30% 이상이 예외면, 원칙보다 예외가 많은 상태입니다. |
| G-01-c | 포괄 예외 | "대통령령으로 정하는 경우 제외"면, 시행령에서 원칙을 무력화할 수 있습니다. |
| G-02-a | 절차 종류 | (정보 수집용) |
| G-02-b | 중복 절차 | 같은 일에 허가+등록+신고를 모두 받아야 하면, 시간과 비용이 불필요하게 듭니다. |
| G-02-c | 처리 기한 | 기한이 없으면, 신청 후 언제 결과를 받을지 알 수 없습니다. |
| G-02-d | 간주 규정 | 기한이 지나도 간주 규정이 없으면, 기관이 무한정 미룰 수 있습니다. |
| G-03-a | 감독 범위 | "감독한다"만 있고 뭘 보는지 없으면, 감독이 형식에 그칩니다. |
| G-03-b | 감독 주기 | 주기가 없으면, 감독을 안 해도 의무 위반이 아닙니다. |
| G-03-c | 감독 방법 | 서면/현장 구분이 없으면, 서류만 보고 넘어갈 수 있습니다. |
| G-03-d | 결과 공개 | 결과를 공개 안 하면, 감독이 실제로 이뤄졌는지 외부에서 확인할 수 없습니다. |
| G-03-e | 시정 명령권 | 문제를 발견해도 고치라고 할 권한이 없으면, 발견만 하고 끝납니다. |
| G-04-a | 통제환경 | 누가 무엇을 책임지는지 정해지지 않아, 문제가 생겨도 책임 소재를 따질 수 없습니다. |
| G-04-b | 위험평가 | 어떤 업무에 어떤 위험이 있는지 파악하지 않아, 사고가 터진 후에야 문제를 인지하게 됩니다. |
| G-04-c | 통제활동 | 승인 절차나 업무 분리가 없어, 한 사람이 결정부터 집행까지 할 수 있습니다. |
| G-04-d | 정보소통 | 현장 정보가 경영진에게 전달되는 체계가 없어, 문제를 알고도 보고되지 않을 수 있습니다. |
| G-04-e | 모니터링 | 스스로 점검하는 절차가 없어, 규정을 어겨도 내부에서 발견할 수 없습니다. |
| G-05-a | 보고 주기 | 주기가 없으면, 보고를 언제 해야 하는지 기준이 없습니다. |
| G-05-b | 보고 양식 | 양식이 없으면, 담당자마다 다른 형태로 보고하여 비교·분석이 불가능합니다. |
| G-05-c | 보고 방법 | 전자적 방법 규정이 없으면, 서면 보고만 가능하여 속도가 느립니다. |
| G-05-d | 미보고 제재 | 안 해도 벌이 없으면, 보고하지 않을 유인이 큽니다. |

### 10.6 효율성 (E) 서브체크 사유

| ID | 서브체크 | 사유 |

| E-01-a | 조건 단계 | 5단계 이상 조건을 모두 충족해야 하면, 담당자도 판단을 틀리기 쉽습니다. |

| E-02-a | 서식 수 | 서식이 20종 이상이면, 어떤 서식을 써야 하는지 찾는 것 자체가 일입니다. |
| E-02-b | 변경 절차 | 서식 하나 바꾸려면 시행규칙을 개정해야 하면, 변경에 수개월이 걸립니다. |

| E-03-a | 서면 전용 | "서면으로"만 허용하면, 온라인으로 처리할 수 없습니다. |
| E-03-b | 날인·인감 | 날인이 필수면, 비대면 처리가 원천 차단됩니다. |
| E-03-c | 대면 전용 | 직접 출석해야 하면, 시간·비용이 불필요하게 듭니다. |
| E-03-d | 우편 전용 | 등기우편만 허용하면, 전자 송달보다 며칠이 더 걸립니다. |
| E-04-a | 유형 수 | 규제 대상이 7유형 이상이면, 조문마다 "이 유형은 해당되나" 확인해야 합니다. |
| E-04-b | 차등 근거 | 왜 차등하는지 근거가 없으면, 자의적 차별로 다툼이 생깁니다. |
| E-04-c | 관통 복잡도 | 유형 구분이 법 전체를 관통하면, 모든 조문을 유형별로 읽어야 합니다. |
| E-05-a | 의무-벌칙 매칭 | 의무는 있는데 벌칙이 없으면, 안 지켜도 불이익이 없습니다. |
| E-05-b | 제재 수위 | 중대한 의무인데 제재가 가벼우면, 위반 비용이 이행 비용보다 낮습니다. |
| E-05-c | 정책 제외 | "노력하여야 한다"는 법적 의무가 아니라 훈시 규정이므로 제재공백이 아닙니다. |
