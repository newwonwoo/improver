# "70점" 도달 가능성 분석 — 정직한 결론 (2026-06-13)

요구: 결함 탐지(원인 짚기) 능력이 **최소 0.70**이어야 쓸모 있다.
현재: 프로덕션 앙상블 0.539 / OOF 스태킹 0.575. → 70까지 +0.12 이상 필요.

## 1. 모델 측 한계 시도 (전부 측정)
| 시도 | 결과 |
|---|---|
| 꺼진 텍스트 신호 활성화 | struct 0.500 → text 0.556 |
| OOF 스태킹 + 보정 | **0.575** (전 단일모델 상회) |
| rule_id 특징 추가 | 효과 거의 없음(0.574) |
| Gradient Boosting + 텍스트 SVD | 효과 거의 없음(하이브리드 0.574) |

→ **모델 잔기술로는 ~0.57~0.60이 천장.** 더 큰 NN(BERT/GNN)은 소표본 오버피팅.

## 2. 천장의 진짜 원인 — 라벨/기준의 주관성 (핵심)
**강한 LLM(Opus)에게 동일 조문 60건(TP30/FP30 균형)을 읽혀 사람 정답과 비교:**
- 일치율 **0.52** (균형 데이터에서 거의 동전던지기)
- precision 0.55 / recall 0.20 / F1 0.293

즉 **최고 수준 AI가 조문을 직접 읽어도 사람 라벨과 절반만 일치**한다.
"이게 고쳐야 할 결함인가"의 경계가 주관적이거나, 조문 발췌만으론 판정 불가
(예: 인용 유효성은 인용된 법을 일일이 확인해야 함). 즉 **0.575라는 모델 점수는
이미 라벨 잡음이 만든 천장에 근접**해 있다.

## 3. 사실상 망가진 룰 (거의 전부 헛경보)
| 룰 | 표본 | TP율 |
|---|---|---|
| S-01 (삽입조 과다) | 94 | 0% |
| S-02 (위임 검증) | 92 | 0% |
| L-03 (참조 끊김) | 294 | 1% |

이 3룰은 발화의 99~100%가 헛경보 → 탐지 정밀도를 통째로 끌어내림.

## 4. 결론 — 70은 모델이 아니라 "기준"을 고쳐야 도달
1. **결함 정의를 객관적으로 결정 가능하게 날카롭게**: 예) S-04 "열거 과다"를
   "이질적 항목 ≥15개 AND 의무유형 혼재"처럼 명확 기준으로. 그래야 사람·모델이 일치.
2. **망가진 룰(S-01/S-02/L-03) 폐기 또는 재설계** → 헛경보 대량 제거(정밀도 즉시 상승).
3. **명확 기준 하에 재라벨링**(가이드라인 통일) → 그 후에야 모델·LLM이 고일치 가능.
4. **의미 판단이 본질인 룰**(F-01/F-03/L-01)은 LLM 판정관을 루프에 넣는 구조 검토.
   단, LLM도 기준이 모호하면 못 맞춤(위 52%) → 1번이 선행 조건.

## 5. 이미 깨끗한(읽으면 바로 보이는) 결함 — 지금도 고정밀 가능
- **G-03**: "○○는 △△가 감독한다" 한 줄짜리 감독규정(보고·검사·시정 권한 부재)
- **E-03**: 전자 대안 없는 날인·서면 강제

→ 이런 '객관적으로 결정 가능한' 결함부터 룰을 좁히면 해당 룰 정밀도는 70+ 가능.
