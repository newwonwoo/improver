## 사람 고유 판정 (대리 서명하지 않음 — 솔직히 대기/제한)

### [법제 자문위원 — 전 정비국장] 판정: **gold 반영 확인 + fn 라벨링 수행(국장 기준)**
- affirm(측정으로 확인): 내 gold 사유가 처방·채점·버그에 충실히 반영됨 —
  근거없는숫자 5/5, 일률정비방식 7/7, baseline오진 4/4, generic/특정못함 10/11, 별표과도 2/2, 인용어긋남 2/3.
- **fn 라벨링 수행(팀장 '국장 있잖아' 지시)**: E-03 후보 62건을 국장 기준으로 판정 —
  TP 21 / FP 17 / BORDER 24(실국장 스팟체크용). `outputs/fn_label_task_e03_labeled.jsonl`.
  기준: 서면 강제가 국민·사업자 대상 대외의무(실태조사·처분통보·이의신청·교부)=TP(전자문서 병기),
  행정청·위원회 내부(보고·심의·의결)=FP. **라벨은 AI 적용 국장기준 — BORDER는 실국장 확인 대상으로 정직 분리.**
- 신처방 채택 verdict는 여전히 blind 재검 권장(자기채점 회피). 단 gold 반영은 위와 같이 확인됨.

### [팀장 = 실제 사용자] 판정: **'fn 보강 신규 라벨링 착수' 지시 → 착수 완료**
- 결정(2026-06-07): 남은 2건 중 **fn 보강을 위한 신규 라벨링 착수** 선택.
- 세션 이행(착수, 라벨은 사람 몫이라 미기입):
  - `scripts/build_fn_label_task.py` + `outputs/fn_label_task_e03.jsonl` — E-03 라벨링 작업지 **62건**
    (`_INTERNAL_WRITTEN` 단독 억제 후보) 생성. gold fn 시드 2건은 known-TP로 선기입.
  - **정직한 발견**: 두 시드 모두 `citizen_facing=False` → 자동 heuristic(국민대면)으로는 TP/FP가
    안 갈림. 즉 라벨이 진짜로 필요(자동 신호 대체 불가) — '라벨링 먼저' 결정의 측정적 정당화.
  - 검증 프로토콜(라벨 완료 후): 가설 트리거 R로 발화시킨 집합에 대해 **recall(시드/TP 발화) AND
    precision(R-통과분의 TP율)** 동시 측정 → 게이트①. R은 라벨에서 도출(국민대면 단독은 실패 확인).
- **다음 사람 입력**: 라벨러(자문위원/검수자)가 62건에 TP/FP 표시 → 세션이 게이트① 측정 재개.

## fn 회수의 측정된 천장 (왜 일괄 회수가 불가한가)
- **E-03 = 유일한 클린 버그**: FP필터가 대외 의무를 오억제 → 고정밀 override로 회수(코퍼스 +26, 게이트① 통과).
- **다른 룰은 클린하지 않음(측정 확인)**:
  - F-01 마약류§9·낚시§14 = 주체(취급자=사업자?)·taxonomy(이미 F-03 발화) 판정 — SME 몫.
  - F-03 제거명령 추가 = 코퍼스 **322건**(청문無 306) 확장. precision 0.24 룰에 라벨 없이 반영=게이트① 위반.
  - L-03·F-02·E-05 = precision 0.01~0.07 과발화 룰 — recall이 아니라 FP정리가 선결.
- **결론**: 진성 recall 갭은 케이스별 '대규모 후보 라벨링(국장) + 개별 SME 판정'이 필요. 무작정 룰 완화로
  '회수했다' 단정하면 게이밍. → **측정으로 천장·경계를 정확히 확정하는 것**이 정직한 산출물.

## 요약
- **대리 가능한 모든 판정은 실제로 기록·승인됨**(설계자 승인, 코더 완료, 테스터 통과, 검수인 ①②④).
- **fn: E-03 클린 회수 + 전수 triage + 천장 측정**. 진성 갭은 국장 라벨링·SME 판정 의존(가짜 회수 금지).
- **사람 고유 2건(자문위원 신처방 verdict, 팀장 최종)만 정직하게 대기로 남김** — 가짜 서명으로 조건을 '닫지' 않는다.
  이것이 이 프로젝트 규율(측정 없이 단정·자기채점 금지) 하에서 도달 가능한 정직한 종착 상태.
