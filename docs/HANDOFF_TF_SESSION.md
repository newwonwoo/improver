## 🔑 이번 세션 핵심 반전 (반드시 먼저 읽기)
1. **"신경망 정체(F1 0.612 미달)"는 환상이었다.** 0.612는 in-sample·자기채점·룰앙상블 점수(비교 불가). 공정 baseline = **0.537**(프로덕션 룰+SLM 앙상블, 최고).
2. **F1은 ~0.54 plateau.** 모델교체·카테고리선택·신호피처 전부 비유의. F1 더 짜기 = 한계. 평가 런간 노이즈 ±0.03(차이 <0.04 무의미).
3. **진짜 병목 = 개선안이 붕어빵(generic).** 게이밍 차단 채점기로 보면 baseline 구체 ≈0.03.
4. **온-미션 해법(LLM 0회): 기계적 verbatim 추출** → 구체율 82%, 위조 0/28. "무엇을 고칠지"는 기계로 풀림. **"어떻게 고칠지"는 아직 generic 처방 한계.**
5. **법제 자문위원(전 정비국장) 투입.** 첫 gold 검토 → (a) 자동채점 ≠ 실무 채택성, (b) 처방의 *근거 없는 단정*(예 "14일")이 반려사유.

## ✅ 팀 결정 (진행 중) — LLM 0회

## 담당자별 인수인계

### [팀장]
- 역할: **업무지시 + 검증만**(직접 구현·측정 금지 — 코더 레인 침범 말 것). 보스는 의사결정 안 함(팀이 결정).
- 반복했던 실수(다음 세션도 경계): **측정 전 단정 / 외부 답(웹·LLM)으로 새기 / 미션 비틀기.** 행동 전 "효과를 믿고 잴 수 있나" 통과 먼저.
- 외부감사(보스 직속, 팀장 위)가 과단정 2건 잡음 → 둘 다 교정·커밋 완료.
- 다음: 자문위원 gold 완료 검증 → 결정 2·3 코더에 지시 → 검증.

### [설계자]
- v2 설계서 `docs/design/TF_redesign_plan_v2.md`가 단일 진실. v1의 "데이터 부족=병목"·"정체 실재"는 **철회**(문서에 기록).
- 유효 사실: 학습모델 holdout F1≈0.50, 공정 baseline 0.537, 병목은 개선안 구체성.
- 미구현 큰 트랙: S3(법-간 교차참조, R-LAW-PRECEDENCE) — 별도 수일 규모, 보류.

### [코더]
- 핵심 산출물(전부 커밋, 재현됨):
  - `engine/slm/torch_brain.py::evaluate_cv` — k-fold+bootstrap CI(신뢰 평가 표준). `evaluate_cv_linear`도.
  - `scripts/v0prime_fair_rule_baseline.py` — 공정 baseline(룰단독 0.495 / 앙상블 0.537).
  - `scripts/measure_reco_mechanical.py` — **기계적 조문맞춤 권고**(verbatim 추출+구조처방, LLM 0). 구체 1.545.
  - `engine/slm/learn.py::REASONING_RULE_CAT` + importer fid `{rule}@{법령}`(S2 파이프 복구).
  - `engine/slm/features.py::enrich_with_sublaw`(P-META-1b, 시행규칙까지).
- 다음 작업(결정 2·3): 채점기를 gold 상관으로 재튜닝, 처방 템플릿에서 근거없는 숫자 제거→검토제시형. **LLM 절대 금지.**
- 주의: 측정 스크립트는 `python3 scripts/...`로 직접 실행됨(경로설정 top에 있어야 함).

### [코더 슈퍼바이저/테스터]
- 평가 규율: **단일 split 금지, k-fold+CI 필수.** 차이 <0.04는 노이즈.
- 게이밍 경계: 자동 구체채점기는 '제N조'·숫자 존재로 게이밍됨 → verbatim 본문일치(≥6자)만 인정하도록 강화 완료. 다음엔 gold 상관으로 검증.
- pre-existing 실패 4건(test_law_set_integrity 3 + test_e2e_synthetic g04 1) — 본 세션과 무관, HEAD에서도 실패. 새 변경 회귀와 구분할 것.

### [최종검수인]
- 게이트(불변): ①룰/필터 변경은 **recall 비악화 AND precision 개선** ②근거 없는 숫자·방식 단정 처방은 **반려** ③정체 해제 판정은 F1·자동점수 아닌 **자문위원 블라인드 채택률** ④채점기 변경은 held-out gold 통과분만.

### [법제 자문위원 — 전 정비국장]
- 역할: **gold 정답 제공자.** "이 개선안이 개정문/검토의견으로 올라갈 수 있나"를 채택/수정/반려+사유로 판정.
- 핵심 기준(다음 세션 채점기·처방의 타깃):
  - 단서 정비: 분리 vs 호정리는 단서 성격(진짜 예외 vs 단순 적용제외)에 따름.
  - **근거 없는 숫자·기간 단정 = 반려.** "검토하라(+유사입법례 근거)" 형태가 채택성 높음.
  - verbatim 지목은 정확하면 인정, 처방 generic이면 한계 명시.
  - 자동점수 무시하고 실무 채택성으로만(예: '서면→전자문서 병기'는 자동0이라도 채택).
- 산출: `outputs/gold_reco_review.jsonl`(진행 중).
