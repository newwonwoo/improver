# 팀원회의록 — 사회 보편 인식 반영 방안 (사회학자 교수 초빙)

> 소집: 부서장. 일자 2026-06-10. 안건: "범용 규정개선기로서 대한민국 사회의
> 보편적 인식을 반영할 방법이 있는가? 방안 도출·반영."
> 참석: 부서장(좌장), 초빙 사회학자 교수, 설계자, 코더, 테스터, 검수인,
> 법제 자문위원(전 정비국장), 감사인.

## 1. 문제 정의 (부서장)

현 엔진의 결함 판정은 **법리(法理) 내부 기준**(포괄위임·청문누락·과도재량 등)에만
근거한다. 그러나 '규제 정비'의 본래 수요는 **국민이 실제로 불편·부당하다고 느끼는
지점**이다. 법리상 합법이어도 사회적으로 정비 요구가 큰 조문이 있고, 그 역도 있다.
→ 사회 인식을 어떻게 **측정 가능한 신호**로 들여올 것인가.

## 2. 초빙 사회학자 교수 의견 (핵심)

- **함정 경고**: "여론을 직접 라벨로 쓰면 다수결 편향·시의성 편향에 오염된다.
  사회 인식은 **정답(label)이 아니라 우선순위(priority) 신호**로만 써야 한다."
- **타당한 구성개념(construct)**: 사회 인식은 세 축으로 조작화(operationalize) 가능 —
  1. **현저성(salience)**: 특정 규제 주제가 공론장에서 얼마나 자주·강하게 거론되는가
     (뉴스·민원·지식iN 빈도). → 정비 *우선순위* 가중.
  2. **체감 부담 방향(valence)**: 그 거론이 '불편·과도·부당' 쪽인가 '보호 필요' 쪽인가.
     → 결함 *방향* 검증(엔진이 결함이라 본 게 사회 체감과 일치하는가).
  3. **수범자 영향 범위(reach)**: 일반 국민·소상공인 대상인가, 행정 내부인가.
     → 이미 엔진에 `citizen_facing` 신호 존재(e03_analog) — 사회적 가중의 앵커.
- **방법론 권고**: "표준화된 **사회 현저성 지수(Social Salience Index, SSI)**를 조문
  주제별로 만들어, 결함 *점수에 곱하지 말고*(왜곡) **정비 우선순위 정렬과 리포트
  '사회적 맥락' 절에만** 쓰라. 탐지 정확도(F1)와 사회 가중을 분리하라."
- **편향 통제**: 출처 다변화(보수·진보 매체 혼합), 기간 윈도우 고정, 검색어를
  법령 주제어에서 기계 생성(체리피킹 금지), 원자료·쿼리 전량 보존.

## 3. 역할별 검토

- **설계자**: SSI를 점수 곱셈에 넣으면 §6(메트릭 게이밍) 위반. **분리 설계 동의** —
  탐지층(법리)과 우선순위층(사회)을 2-레이어로. 리포트에 '사회적 맥락' 절 신설.
- **코더**: 실현 수단 이미 있음 — NaverSearch(news/kin) MCP + `save_naver_results.py`
  파이프라인 + `rule_mining/sources/crawled/naver/`. 주제어→검색→빈도/감성 집계까지
  로컬로 가능. LLM 없이 사전기반 valence(불편/부담/과도 vs 보호/안전) 1차 산정.
- **테스터**: SSI는 **정답이 없는 지수** → F1로 검증 불가. 대신 **수렴타당도**로 검증:
  '국무조정실/지자체가 실제 정비한 규제 목록'과 SSI 상위가 겹치는지(rank correlation).
  과적합 방지 위해 구축기간과 검증기간 분리.
- **검수인**: 게이트 추가 — (가) SSI는 탐지 F1에 영향 0임을 측정으로 보일 것(같은
  F1, 정렬만 변화). (나) 출처·쿼리·원문 전량 보존. (다) 사전(lexicon)은 공개·재현가능.
- **법제 자문위원**: "법리 결함이 없는데 여론만 높다고 '결함' 찍으면 안 된다. SSI는
  **이미 탐지된 결함들 사이의 우선순위**와 **놓친 영역 탐색 힌트**로만." 동의.
- **감사인**: 승인하되 4개 조건 — ①SSI는 라벨·학습·F1에 유입 금지(우선순위/리포트 전용)
  ②출처 다변화·기간고정·쿼리 기계생성 ③원자료 전량 보존 ④수렴타당도로 사후 검증.

## 4. 도출 방안 (회의 합의) — 사회 현저성 지수(SSI) 2-레이어

```
[레이어 1] 법리 탐지 (현행 유지, F1 불변)
    조문 → 룰/SLM → 결함 findings (카테고리·등급)

[레이어 2] 사회 우선순위 (신규, 분리)
    조문 주제어 추출 → NaverSearch(news+kin, 기간고정)
      → salience = log(1+빈도)  (현저성)
      → valence  = 사전기반 (불편/부담/과도/규제완화 vs 보호/안전/강화)
      → reach    = citizen_facing 신호 연계
    SSI = z(salience) 가중, valence 로 결함방향 일치여부 플래그
    → 리포트 '사회적 맥락' 절 + 정비 우선순위 정렬 (점수 비곱셈)
```

검증: 실제 정비 규제 목록과 SSI 상위 rank correlation(수렴타당도). 게이밍 0 입증
(SSI on/off 시 탐지 F1 동일, 정렬만 변화).
